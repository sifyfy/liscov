# ADR-002: 認証状態インジケーターの追加

## ステータス

承認

## コンテキスト

従来、認証状態は設定画面を開かないと確認できなかった。
セッション切れが発生しても通知がなく、ユーザーは配信接続に失敗して初めて認証問題に気付く状況だった。

また、「自動再認証」機能の要否を検討した結果、インジケーターで状態が分かれば手動での再ログインで十分と判断した。

## 決定

アプリ右上（ヘッダー領域）に認証状態インジケーターを常時表示する。

## 理由

### 検討した選択肢

| 選択肢 | メリット | デメリット |
|-------|---------|-----------|
| 自動再認証 | ユーザー操作不要 | 実装複雑、意図しないタイミングでウィンドウが開く可能性 |
| インジケーター表示 | 状態が常に見える、実装シンプル | 手動操作が必要 |
| 通知ダイアログ | 問題発生時に気付ける | 鬱陶しい、作業の邪魔になる可能性 |

### 採用理由

- インジケーターがあれば、ユーザーは必要なタイミングで自主的に再ログインできる
- 自動再認証は予期しないタイミングでウィンドウが開くため、配信中などに邪魔になる可能性
- シンプルな実装で目的を達成できる

## 影響

- **新規コンポーネント:** `AuthIndicator.svelte`
- **新規コマンド:** `auth_check_session_validity`
- **新規イベント:** `auth-session-validity-changed`
- **レイアウト変更:** ヘッダー領域にインジケーター配置スペースが必要
